⏳ Usage-Limit Rescheduler

tasks that die on an API limit come back on their own — right after the window resets

How it works

🔍Detecta task lands in failed; the monitor agent reads its session output for limit signatures (429, “usage limit”, “resets at …”)
🕐Wait it outreset time from the output, the Anthropic usage API (admin key), or a +60 min fallback — booked as a one-time schedule
▶Restartwhen the schedule fires, the task relaunches automatically — up to 3 attempts, then it gives up loudly
The detection policy is plain instructions on this addon's agent — open its Customize chat to tune it (different backoff, more attempts, other failure texts).
